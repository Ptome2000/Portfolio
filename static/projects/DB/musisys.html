Database Musisys Project

Description

This project is an advanced database designed to facilitate the robust management of music festivals. It handles a wide range of data related to festival editions, stages, participants, technicians, spectators, and ticketing.

The project aims to streamline festival organization, enhance operational efficiency, and ensure a comprehensive record-keeping process for all festival-related activities.

Key features include:
  • Track festival editions with unique identifiers, names, locations, venues, dates, and capacity
  • Manage up to seven stages per festival, each with a unique code and name
  • Register and track stage setup technicians and participant-associated roadies
  • Record participant details, performance times, styles, fees, song sequences and guest appearances
  • Identify spectators as journalists, guests, or paying spectators, recording specific attributes for each category
  • Manage ticket sales and distributions, including single-day, multi-day tickets and the return of these with transaction logging
  • Schedule and record interviews between journalists and participants
  • Optimized relational database for enhanced performance and efficiency
  • Automated processes for routine tasks to streamline operations and reduce manual effort (Triggers, Stored Procedures and Functions)
  • Implement a set of advanced search functionalities to facilitate quick and accurate retrieval of information with Queries and Views
  • Demonstration prototype using HTML and PHP to showcase the system's features and functionalities in a web-based interface
